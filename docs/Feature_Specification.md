## 기능 A. 사용자 인증 (로그인/회원가입)

### 1. 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 사용자 인증 (로그인/회원가입) |
| 기능 ID | FR-AUTH |
| 역할 | 앱 최초 실행 시 사용자를 등록·인증하고, 이후 자동 로그인을 유지하여 응급 상황에서 불필요한 인증 단계를 제거한다. 사용자 유형(일반/구급대원)을 구분하여 맞춤 기능을 제공한다. |
| 대상 사용자 | 전체 (일반 사용자 + 구급대원) |

### 2. 사용자 시나리오

| 시나리오 | 상세 흐름 |
|----------|-----------|
| **최초 회원가입** | 1. 사용자가 앱을 처음 설치·실행한다.<br>2. 회원가입 화면이 표시된다.<br>3. 이메일, 비밀번호, 닉네임을 입력한다.<br>4. 사용자 유형을 선택한다 (일반 사용자 / 구급대원).<br>5. 구급대원 선택 시 소속기관 코드를 추가 입력한다.<br>6. 회원가입 완료 후 자동 로그인되어 메인 대시보드가 표시된다. |
| **자동 로그인** | 1. 사용자가 앱을 재실행한다.<br>2. 저장된 인증 토큰이 유효하면 로그인 화면을 건너뛴다.<br>3. 바로 메인 대시보드(응급실 검색 화면)가 표시된다. |
| **재로그인** | 1. 토큰 만료 또는 로그아웃 후 앱 실행 시 로그인 화면이 표시된다.<br>2. 이메일/비밀번호 입력 후 로그인한다. |

### 4. 데이터 흐름

| 구분 | 필드명 | 타입 | 필수 | 설명 |
|------|--------|------|------|------|
| **입력** | email | String | ✅ | 사용자 이메일 (로그인 ID) |
| **입력** | password | String | ✅ | 비밀번호 (해시 저장) |
| **입력** | nickname | String | ✅ | 사용자 닉네임 |
| **입력** | user_type | Enum | ✅ | `GENERAL` / `PARAMEDIC` |
| **입력** | org_code | String | 조건부 | 구급대원 소속기관 코드 |
| **출력** | access_token | String | — | JWT 인증 토큰 |
| **출력** | refresh_token | String | — | 토큰 갱신용 |
| **출력** | user_profile | Object | — | 사용자 프로필 정보 |

**DB 테이블: `users`**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID (PK) | 사용자 고유 ID |
| email | VARCHAR(255) UNIQUE | 이메일 |
| password_hash | VARCHAR(255) | 해시된 비밀번호 |
| nickname | VARCHAR(50) | 닉네임 |
| user_type | ENUM('GENERAL','PARAMEDIC') | 사용자 유형 |
| org_code | VARCHAR(20) NULL | 소속기관 코드 |
| created_at | TIMESTAMP | 생성일시 |
| last_login_at | TIMESTAMP | 최종 로그인 일시 |

### 5. API 및 연동 정보

| API | 메서드 | 엔드포인트 | 설명 |
|-----|--------|-----------|------|
| 회원가입 | POST | `/api/v1/auth/register` | 신규 사용자 등록 |
| 로그인 | POST | `/api/v1/auth/login` | 이메일/비밀번호 인증, 토큰 발급 |
| 토큰 갱신 | POST | `/api/v1/auth/refresh` | 만료 토큰 갱신 |
| 로그아웃 | POST | `/api/v1/auth/logout` | 토큰 무효화 |
| 외부 연동 | — | 없음 | 자체 인증 시스템 |

### 6. 예외 처리 및 에러 케이스

| 에러 상황 | 에러 코드 | 대응 방법 |
|-----------|-----------|-----------|
| 이메일 형식 오류 | AUTH-001 | 입력 필드 하단에 "올바른 이메일을 입력하세요" 표시 |
| 중복 이메일 | AUTH-002 | "이미 등록된 이메일입니다" 안내 |
| 비밀번호 불일치 | AUTH-003 | "비밀번호가 일치하지 않습니다" + 비밀번호 찾기 안내 |
| 토큰 만료 | AUTH-004 | 자동 refresh 시도 → 실패 시 로그인 화면으로 이동 |
| 네트워크 오류 | AUTH-005 | "네트워크 연결을 확인하세요" 팝업 + 재시도 버튼 |
| 구급대원 기관코드 불일치 | AUTH-006 | "유효하지 않은 기관코드입니다" 안내 |

## 기능 B. 환자 상태 입력

### 1. 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 환자 상태 입력 |
| 기능 ID | FR-INPUT |
| 역할 | 응급 환자의 증상, 기본 정보(성별/연령/의식 상태)를 텍스트 또는 음성으로 빠르게 입력받아 적합한 응급실 검색의 조건으로 활용한다. |
| 대상 사용자 | 전체 (일반 사용자 + 구급대원) |

### 2. 사용자 시나리오

| 시나리오 | 상세 흐름 |
|----------|-----------|
| **텍스트 입력** | 1. 메인 화면에서 "응급실 찾기" 버튼을 누른다.<br>2. 증상 카테고리 버튼(두통/흉통/외상/호흡곤란/복통/기타) 중 선택한다.<br>3. 상세 증상을 텍스트로 추가 입력할 수 있다.<br>4. 환자 기본 정보(성별, 연령대, 의식 상태)를 선택한다.<br>5. "검색" 버튼으로 응급실 탐색을 시작한다. |
| **음성 입력** | 1. 입력 화면에서 마이크 아이콘을 누른다.<br>2. "흉통이 있고 호흡이 곤란합니다"와 같이 음성으로 말한다.<br>3. STT 변환 결과가 텍스트로 표시된다.<br>4. 사용자가 확인/수정 후 검색을 실행한다. |
| **구급대원 입력** | 1. 구급대원 모드에서는 추가 항목(활력징후, 외상 부위 등)을 입력할 수 있다.<br>2. 입력 데이터가 자동으로 업무 로그에 기록된다. |

│  ┌────────────────────┬────┐ │
│  │ 텍스트 입력...      │ 🎤 │ │

│                              │

│                              │
│  ┌──────────────────────────┐│

### 4. 데이터 흐름

| 구분 | 필드명 | 타입 | 필수 | 설명 |
|------|--------|------|------|------|
| **입력** | symptom_categories | Array<String> | ✅ | 증상 카테고리 (복수 선택) |
| **입력** | symptom_detail | String | ❌ | 상세 증상 텍스트 |
| **입력** | voice_input_raw | String | ❌ | 음성 인식 원본 텍스트 |
| **입력** | patient_gender | Enum | ✅ | `MALE` / `FEMALE` |
| **입력** | patient_age_group | Enum | ✅ | `INFANT`/`CHILD`/`ADULT`/`ELDERLY` |
| **입력** | consciousness_level | Enum | ✅ | `ALERT`/`DROWSY`/`UNRESPONSIVE` |
| **출력** | severity_score | Integer | — | 자동 산출 중증도 점수 (1~5) |
| **출력** | search_params | Object | — | 응급실 검색 조건 객체 |

| 컬럼 | 타입 | 설명 |
|------|------|------|

| symptom_detail | TEXT | 상세 증상 |
| patient_gender | ENUM | 성별 |
| patient_age_group | ENUM | 연령대 |
| consciousness_level | ENUM | 의식 수준 |
| severity_score | INT | 중증도 점수 |
| created_at | TIMESTAMP | 입력 시각 |

### 6. 예외 처리 및 에러 케이스

## 기능 C. 응급실 검색 및 지도 표시

### 1. 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 응급실 검색 및 지도 표시 |
| 기능 ID | FR-SEARCH-MAP |
| 역할 | 사용자 현재 위치 기반으로 인근 응급실을 검색하고, 수용 가능 여부를 색상 마커(🟢수용가능 / 🟠혼잡 / 🔴불가)로 지도 위에 직관적으로 표시한다. |
| 대상 사용자 | 전체 (일반 사용자 + 구급대원) |

### 2. 사용자 시나리오

| 시나리오 | 상세 흐름 |
|----------|-----------|
| **지도 모드 조회** | 1. 환자 상태 입력 후 검색을 실행한다.<br>2. 현재 위치 중심으로 지도가 표시된다.<br>3. 인근 응급실이 색상 마커로 지도 위에 나타난다.<br>   - 🟢 녹색: 수용 가능 (여유 병상 있음)<br>   - 🟠 주황: 혼잡 (병상 부족하나 수용 가능)<br>   - 🔴 빨강: 수용 불가 (만실)<br>4. 마커를 탭하면 병원 요약 정보(병원명, 거리, 전화번호)가 팝업된다.<br>5. 팝업에서 "길안내" 또는 "전화" 버튼을 누를 수 있다. |
| **검색 반경 변경** | 1. 지도 화면 상단의 반경 필터를 탭한다.<br>2. 5km / 10km / 20km 중 선택한다.<br>3. 선택한 반경에 따라 지도 줌 및 마커가 갱신된다. |

### 4. 데이터 흐름

| 구분 | 필드명 | 타입 | 필수 | 설명 |
|------|--------|------|------|------|

| **입력** | user_lng | Double | ✅ | 사용자 경도 |

## 기능 G. 구급대원 업무 로그 기록

### 1. 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 구급대원 업무 로그 기록 |
| 기능 ID | FR-LOG |
| 역할 | 구급대원이 출동~상황 종료까지의 전 과정을 자동/수동으로 기록하고, 상황 종료 후 업무 처리용 문서(PDF/Excel)로 다운로드할 수 있는 기능을 제공한다. |
| 대상 사용자 | 구급대원 전용 |

### 2. 사용자 시나리오

| 시나리오 | 상세 흐름 |
|----------|-----------|
| **상황 기록** | 1. 구급대원 모드로 로그인한다.<br>2. "출동 시작" 버튼을 누르면 자동으로 타임스탬프가 기록된다.<br>3. 환자 상태 입력 시 데이터가 자동으로 로그에 포함된다.<br>4. 응급실 검색·선택 과정이 자동 기록된다.<br>5. 수용 병원 도착 후 "상황 종료" 버튼을 누른다.<br>6. 총 처리 시간이 자동 산출된다. |
| **문서 다운로드** | 1. "업무 기록" 메뉴에서 이전 출동 기록 목록을 확인한다.<br>2. 원하는 기록을 선택하여 상세 내용을 확인한다.<br>3. "다운로드" 버튼으로 PDF 또는 Excel 파일을 저장한다. |

## 기능 H. 음성 입력

### 1. 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 음성 입력 (Speech-to-Text) |
| 기능 ID | FR-VOICE |
| 역할 | 응급 상황에서 손이 자유롭지 않을 때, 음성으로 환자 상태를 입력할 수 있게 한다. 음성 인식 결과를 텍스트로 변환하여 증상 입력 필드에 자동 반영한다. |
| 대상 사용자 | 전체 (일반 사용자 + 구급대원) |

### 2. 사용자 시나리오

| 시나리오 | 상세 흐름 |
|----------|-----------|
| **음성 입력** | 1. 환자 상태 입력 화면에서 🎤 마이크 버튼을 누른다.<br>2. "말씀하세요..." 안내와 함께 음성 녹음이 시작된다.<br>3. 사용자가 "60대 남성, 흉통이 있고 호흡이 어렵습니다"라고 말한다.<br>4. 실시간으로 텍스트가 변환되어 화면에 표시된다.<br>5. 녹음이 끝나면 변환된 텍스트를 확인/수정할 수 있다.<br>6. "확인" 시 증상 카테고리 자동 매핑도 시도한다. |

│  │ "60대 남성, 흉통이 있고   ││

│  │                          ││
│  └──────────────────────────┘│
