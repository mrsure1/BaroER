# 바로응급실 (BaroER) - 프로젝트 발표 슬라이드 기획안

발표 슬라이드 3~5장에 들어갈 핵심 내용을 구성한 초안입니다. 디자인 및 템플릿에 맞추어 내용을 복사/요약하여 사용하시기 바랍니다.

---

## [Slide 1] 프로젝트 개요: 바로응급실(BaroER) 소개
**부제: 응급 상황의 골든타임을 지키는 실시간 응급실 탐색 서비스**

*   **문제 제기 (Background):**
    *   응급 환자 발생 시, 인근 응급실의 수용 불가(만상)로 인해 여러 병원에 전화를 돌리며 환자 이송이 지연되는 문제(응급실 뺑뺑이)가 빈번하게 발생하고 있습니다.
*   **해결 방안 (Solution):**
    *   실시간 응급실 병상 정보 및 수용 여부를 즉각적으로 확인하고, 환자 상태에 맞는 최적의 병원으로 신속하게 안내하는 웹/앱 통합 서비스(PWA)입니다.
*   **프로젝트 목표:**
    *   환자 탐색 단계의 시간을 단축시켜 생존율을 높이고, 일반 사용자와 구급대원 모두에게 빠르고 직관적인 이송 결정을 지원합니다.

---

## [Slide 2] 주요 핵심 기능 (Core Features)
**부제: 긴급 상황에 최적화된 직관적이고 빠른 서비스 경험**

*   **실시간 수용 정보 시각화 (지도/리스트):**
    *   내 위치 반경(5/10/20km) 내 응급실을 탐색하고, 수용 가능 여부를 직관적인 색상(🟢가능 / 🟠혼잡 / 🔴불가)으로 지도상에 표시합니다.
*   **환자 상태 입력 및 음성 인식(STT):**
    *   위급 상황에서도 타이핑 없이 음성으로 환자 증상을 빠르게 입력하여 중증도를 자동 분류합니다.
*   **자동 전화 확인 시스템:**
    *   동시에 3~5곳의 응급실에 자동으로 전화를 걸어 수용 여부를 확인하고, 결과를 즉각 화면에 반영하여 탐색 시간을 극단적으로 줄입니다.
*   **내비게이션 즉시 연동:**
    *   선택한 병원으로 카카오내비, T맵 등 외부 내비게이션을 즉시 연결하여 길안내를 시작합니다.

## [Slide 3] 타겟 사용자 및 구급대원 특화 기능
**부제: 일반 시민부터 119 구급대원까지 맞춤형 지원**

*   **일반 사용자 (General Users):**
    *   가족이나 본인의 응급 상황에서 당황하지 않고 가장 가깝고 수용 가능한 병원을 쉽게 찾아갈 수 있는 가이드를 제공합니다.
*   **구급대원 전용 모드 (Paramedics):**
    *   소속기관 인증을 통한 전용 인터페이스 제공.
    *   **업무 로그 자동화:** 상황 시작부터 종료까지의 타임스탬프, 환자 상태, 수용 병원 정보, 총 처리 시간 등을 자동으로 기록합니다.
    *   **보고서 내보내기:** 기록된 출동 로그를 PDF나 Excel 파일로 다운로드하여 행정 업무 부담을 대폭 감소시킵니다.

## [Slide 4] 기술 스택 및 아키텍처 (Tech Stack)
**부제: 안정적이고 빠른 서비스를 위한 최신 웹 기술 도입**

*   **플랫폼 환경 (Progressive Web App):**
    *   Next.js 프레임워크 기반으로 개발하여 별도 스토어 설치 없이도 앱처럼 동작하며, Vercel 글로벌 엣지 네트워크를 통해 24시간 365일 높은 가용성을 보장합니다. (추후 TWA를 통해 Play Store 등재 예정)
*   **백엔드 및 데이터 관리 (Supabase):**
    *   Supabase를 활용해 빠르고 안전한 사용자 인증(Auth)과 관계형 데이터(Postgres)를 관리합니다.
*   **외부 API 연동 및 보안:**
    *   공공데이터 포털의 실시간 응급실 정보 API 및 네이버 지도 API를 활용합니다.
    *   API 키 탈취를 방지하기 위해 중요 호출은 브라우저가 아닌 Next.js API Routes (서버사이드)에서 안전하게 처리합니다.

## [Slide 5] 프로젝트 기대 효과 및 요약
**부제: 데이터 기반 응급 의료 접근성 혁신**

*   **사회적 기대 효과:**
    *   신속한 병원 선정으로 응급 환자의 '골든타임' 확보.
    *   구급대원의 병원 탐색 및 행정 보고 시간을 단축하여 현장 구조 업무에 더욱 집중할 수 있는 환경 조성.
*   **향후 확장 계획:**
    *   IndexedDB 및 Service Worker를 활용한 오프라인/통신 불량 환경 대비 캐싱 고도화.
    *   환자 데이터를 기반으로 한 응급실 수용 예측 AI 모델 확장 검토.
